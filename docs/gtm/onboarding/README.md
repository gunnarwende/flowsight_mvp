# Onboarding-Maschine — Entry Point

> **Was hier lebt:** alles, was nach dem "Ja, ich kaufe" passiert, bis der Kunde
> autonom mit dem System läuft. Die Onboarding-Maschine ist der direkte Anschluss
> an die Pipeline-Maschine (`docs/gtm/pipeline/PIPELINE_BIBLE.md`).
>
> **Warum zentralisiert:** Onboarding ist Founder-Bottleneck. Jeder neue Kunde,
> der schneller + sauberer als der vorherige onboarded werden kann, ist 1× weniger
> Founder-Stunde, die wir je wieder ausgeben müssen.

---

## Wenn du gerade...

| ...einen neuen Kunden onboarden willst | → [`ONBOARDING_BIBLE.md`](ONBOARDING_BIBLE.md) §1 + §3, dann [`customer_runbook_template.md`](customer_runbook_template.md) kopieren nach `docs/customers/<slug>/onboarding_<slug>.md` |
| ...die übergeordnete Architektur nochmal anschauen willst | → [`ONBOARDING_BIBLE.md`](ONBOARDING_BIBLE.md) §2 |
| ...wissen willst was wir vom letzten Kunden gelernt haben | → [`lessons_learned.md`](lessons_learned.md) |
| ...einen High-End Hand-Over machen willst (E-Mail-Template, Handout, GBP-Übergabe) | → [`playbook_6_phases.md`](playbook_6_phases.md) |
| ...vom Bottleneck zur Maschine bauen willst | → [`ONBOARDING_BIBLE.md`](ONBOARDING_BIBLE.md) §5 (Roadmap) |
