## 2. Warum der bisherige Proof nicht ausreicht

### Was stimmt: Die Bilder sind gut

Die 3 Banana-Heros (Doerfler Werkstatt, Leuthold Dorf, Orlandini Werkstatt) sind genuinely verschieden und hochwertig. Kein Handwerker wuerde sagen "das sieht gleich aus." Banana liefert.

### Was nicht stimmt: Die Seite drumherum ist identisch

Ein Besucher, der am Hero vorbeiscrollt, sieht sofort:
1. **Gleiche Service-Cards** — 3-Spalter, gleiche Hoehe, gleiche Icons, gleicher "Mehr →" Link
2. **Gleiche Review-Section** — 3 Karten, gleicher Grauer Hintergrund, gleiche Sterne
3. **Gleicher Scrollrhythmus** — Section, weisser Gap, Section, grauer Gap, Section
4. **Gleiche Interaktion** — Klick auf Card → Overlay. Ueberall. Immer.
5. **Null Ueberraschung** — Nichts bewegt sich, nichts reagiert, nichts lebt

### Was wirklich fehlt (ehrliche Liste)

| Fehlt | Wirkung | Prioritaet |
|-------|---------|-----------|
| **Scroll-Animationen** (Sections faden rein) | Seite fuehlt sich "lebendig" an statt "tot" | HOCH |
| **Verschiedene Service-Layouts** (nicht immer 3-Spalter) | Bricht den staerksten Fingerabdruck nach dem Hero | HOCH |
| **Verschiedene Review-Inszenierung** (Carousel vs. Grid vs. Testimonial-Zitat) | Reviews sehen bei jedem Betrieb anders aus | MITTEL |
| **Farbtemperatur im ganzen Seitenkoerper** (nicht nur Hero) | Warme Site vs. kuehle Site ist spuerbar verschieden | MITTEL |
| **Sticky Mobile CTA** (Anrufen + Melden immer sichtbar) | Conversion-Hebel auf Mobile | MITTEL |
| **Parallax auf Hero** (Bild bewegt sich leicht beim Scroll) | Subtiler Premium-Eindruck | NIEDRIG |
| **Video-Hero** (Kling 3.0 Loop) | Wow-Effekt, aber: niemand in der Branche hat es | OPTIONAL |

## 4. Was "hoechstpersoenlich" realistisch bedeuten darf

### Die harte Wahrheit

"100% individuell" bei Founder <= 10 Min und 3 Sites/Tag ist unmoeglich. Punkt.

"100% individuell WAHRGENOMMEN" bei Founder <= 10 Min und 3 Sites/Tag IST moeglich. Und hier ist der Unterschied:

| Aspekt | Echt individuell | Individuell wahrgenommen |
|--------|-----------------|------------------------|
| Hero-Bild | Fotograf kommt vorbei | **Banana generiert Bildwelt passend zum Betrieb** |
| Texte | Texter interviewt Inhaber 2h | **CC schreibt aus prospect_card — verifizierte Fakten, kein Erfinden** |
| Farben | Designerin entwickelt Farbwelt | **Brand-Color + Farbmodus (warm/cool) = genuegend Variation** |
| Layout | Designer entworfen jede Seite | **3-4 Master-Directions, profilbasiert zugewiesen** |
| Fotos | Fotoshoot beim Betrieb | **Banana + gecrawlte Eigenbilder + echte Review-Fotos** |
| Bewertungen | Manuell kuratiert | **Direkt aus Google Reviews — echt, nicht erfunden** |

### Was ZWINGEND individuell/verifiziert/kuratiert sein muss

| Element | Warum | Quelle |
|---------|-------|--------|
| **Firmenname** | Identitaet | prospect_card (verifiziert) |
| **Adresse + Telefon** | Kontakt muss stimmen | prospect_card (verifiziert) |

| **Gruendungsjahr** | Trust-Signal #1 in der Branche | Website-Crawl (verifiziert) |
| **Google-Bewertungen** | Echt oder gar nicht | Google Places API (automatisch) |
| **Team-Mitglieder** | Nur verifizierte Namen | **NUR wenn verifiziert. Sonst: Section weglassen.** |
| **Hero-Bild** | Muss zum Betrieb passen | Banana (Founder waehlt aus 5 Kandidaten) |
| **Brand-Color** | Identitaet | Aus alter Website extrahiert oder Founder entscheidet |

### Was NICHT mehr diskutiert werden darf

| Element | Entscheidung | Wer entscheidet |

## 5. Architektur der echten Modul-1-Website-Maschine

### Die 8 Ebenen

**Ebene 1: WAHRHEITSKERN (verifizierte Daten)**
Alles was auf der Website steht, muss WAHR sein. Kein Erfinden, keine Halluzinationen. Daten kommen aus prospect_card.json (verifiziert durch Scout + Crawl + Gates V1-V3).

**Ebene 2: IDENTITAETSPROFIL (automatisch zugewiesen)**
3 Profile (TRADITION/NAEHE/KOMPETENZ) steuern ALLES visuelle. Zuweisungslogik:

```
WENN Gruendungsjahr < 1985 UND History >= 3 Eintraege → TRADITION
WENN Reviews >= 4.5 UND Count >= 20 ODER Team <= 2 → NAEHE
SONST → KOMPETENZ
```

**Ebene 3: DRAMATURGIE (profilbasierte Story-Architektur)**
Nicht "Section nach Section" sondern eine gefuehrte Reise:

| Akt | TRADITION | NAEHE | KOMPETENZ |
|-----|-----------|-------|-----------|
| 1 — IMPACT | Werkstatt-Hero + "Seit 1926" | See-Hero + "4.9★ aus 44 Bewertungen" | Praezisions-Hero + "5 Fachbereiche · 17 Partner" |
| 2 — BEWEIS | Geschichte/Timeline | Bewertungen (gross, emotional) | Services (Grid oder Liste) |
| 3 — VERTRAUEN | Services + Team | Einzugsgebiet (Karte) | Partner + Zertifikate |
| 4 — HANDLUNG | Bewertungen + Kontakt + Wizard | Services + Kontakt + Wizard | Karriere + Bewertungen + Kontakt + Wizard |

**Ebene 4: BILDWELT (AI-generiert via Banana)**
Pro Betrieb: 1 Hero-Keyframe + 1 Atmosphaere-Bild (ServiceArea). Optional: Service-Keyvisuals. Founder waehlt Hero aus 5 Kandidaten (30 Sek).

**Ebene 5: MOTION (framer-motion + optional Kling)**
- DEFAULT: Scroll-Animationen (framer-motion, CHF 0, einmal gebaut)
- DEFAULT: Parallax auf Hero (CSS, CHF 0)
- OPTIONAL: Kling 3.0 Hero-Loop (CHF 0.28/Clip, nur bei expliziter Founder-Entscheidung)

**Ebene 6: TRUST-INSZENIERUNG (profilbasiert)**
Welches Trust-Element FUEHRT, haengt vom Profil ab (siehe Dramaturgie oben).

**Ebene 7: WIZARD-/CONVERSION-ARCHITEKTUR**
- Hero: "Anliegen melden" Button (sichtbar, Brand-Color)
- Mobile: Sticky Footer-Bar ("Anrufen" + "Online melden") — immer sichtbar
- Letzte Section: Kontakt + Wizard-CTA + Karte

**Ebene 8: MOBILE-FIRST-ERLEBNIS**
80%+ des Traffics ist Mobile. Alles muss auf 375px Breite ZUERST funktionieren. Desktop ist der Bonus.

## 7. Empfohlene Master-Directions

### Ich empfehle 3 (nicht 4)

4 Directions klingen gut, sind aber operativ problematisch: Die Zuweisungslogik wird unscharf, die Wartung verdoppelt sich, und der vierte Typ ("Premium-Komplettanbieter") ist in der Praxis fast immer KOMPETENZ. Drei reichen.

**Direction A: SUBSTANZ (vorher "Tradition")**

| Aspekt | Auspraegung |
|--------|-------------|
| **Wer** | Betriebe > 40 Jahre, Generationsbetriebe, Geschichte als Staerke |
| **Hero** | Split (Bild rechts, Text links). Warm. Source Serif Headings. |
| **Bildwelt** | Banana: Werkstatt, Handwerk, Patina, Morgenlicht, Holz, Messing |
| **Dramaturgie** | Geschichte → Services → Bewertungen → Team → Kontakt |
| **Animations** | Fade-up (langsam, ruhig, wie ein Album) |
| **Services** | 2-Col mit grossem Bild links (nicht 3-Col Grid) |
| **Reviews** | Einzelzitat gross (1 starkes Review prominent) |
| **Above-the-fold** | "Seit [JAHR]" + Firmenname + warmes Bild + "Anliegen melden" |

**Direction B: VERTRAUEN (vorher "Naehe")**

| Aspekt | Auspraegung |
|--------|-------------|
| **Wer** | Kleine Betriebe (1-5 MA), starke Reviews, lokal verankert |
| **Hero** | Center (zentriert, starker Overlay). Warm. DM Sans. |
| **Bildwelt** | Banana: Dorf, See, Nachbarschaft, goldene Stunde, Geborgenheit |
| **Dramaturgie** | Bewertungen → Einzugsgebiet → Services → Kontakt |
| **Animations** | Slide-in (leicht, natuerlich, wie eine Tuer) |
| **Services** | Stacked List (vertikal, ein Service pro Zeile, gross) |
| **Reviews** | Horizontal Scroll (Carousel mit grossen Karten) |
| **Above-the-fold** | "4.9★ aus 44 Bewertungen" GROSS + Ortsname + See-Bild |

**Direction C: PRAEZISION (vorher "Kompetenz")**

| Aspekt | Auspraegung |
|--------|-------------|
| **Wer** | Groessere Betriebe (5+ MA), breites Spektrum, viele Partner/Zertifikate |
| **Hero** | Classic (Vollbild, Gradient). Cool. Inter. |
| **Bildwelt** | Banana: Moderne Werkstatt, Edelstahl, LED, Praezision, Ordnung |
| **Dramaturgie** | Services → Partner → Karriere → Bewertungen → Kontakt |
| **Animations** | Scale-in (Cards wachsen rein, praezise, kontrolliert) |
| **Services** | 3-Col Grid (aktuelles Layout, passt zu "breit aufgestellt") |
| **Reviews** | Grid (3 Karten, wie aktuell — passt zum professionellen Stil) |
| **Above-the-fold** | "X Fachbereiche · Y Partner" + modernes Bild + kuehle Toene |

## 8. Rollen von CC / Banana / Kling 3.0

### CC = Orchestrator + Code + Qualitaet

CC macht:
- prospect_card validieren (Gates V1-V3)
- Profil zuweisen (Regel-basiert, keine Diskussion)
- CustomerSite Config schreiben (300 Zeilen TS)
- Banana API aufrufen (Hero + Atmosphaere)
- Kling API aufrufen (nur wenn Founder "Go" gibt)
- framer-motion Animationen sind EINMAL gebaut (kein Aufwand pro Betrieb)
- Build + Deploy + Lighthouse-Check
- Wizard-E2E-Test

CC macht NICHT:
- Design-Entscheidungen pro Betrieb (Profil entscheidet)
- Texte umschreiben nach Geschmack (1 Runde, dann Freeze)
- Endlose Feedback-Loops fuehren

### Banana = Statische Bildidentitaet

Banana macht:
- Hero-Keyframe (5 Kandidaten, profilbasierter Prompt, CHF 0.15)
- Atmosphaere-Bild fuer ServiceArea (3 Kandidaten, lokales Szenario, CHF 0.09)
- Service-Keyvisuals wenn gecrawlte Bilder fehlen (CHF 0.09/Service)

Banana macht NICHT:
- Team-Fotos (wirken unecht)
- Logos oder Icons
- Texte
- Videos/Motion (→ Kling)

### Kling 3.0 = Optionale cinematische Differenzierung

Kling macht (NUR auf Founder-Entscheid):
- Hero-Loop (5s, aus Banana-Keyframe, CHF 0.28-0.56)
- Subtile Kamerabewegung + Lichtverschiebung

Kling macht NICHT:
- Standard-Feature bei jeder Website (zu teuer in der Wartung, Ladezeit)
- Service-Clips (Overkill)
- Ganze Videosequenzen

**Meine ehrliche Einschaetzung zu Kling:**
Kling ist ein "nice-to-have" fuer die Website, kein "must-have". Die Branche hat KEINE Video-Heros. Der Wow-Effekt ist real, aber der ROI ist fragwuerdig fuer CHF 299/Mo Kunden. Empfehlung: Kling fuer die EIGENE Marketing-Website (flowsight.ch) einsetzen. Fuer Kunden-Websites: nur auf ausdruecklichen Founder-Wunsch.

### Was bewusst NICHT AI-generiert werden darf

| Element | Warum |
|---------|-------|
| **Alle Fakten** (Name, Adresse, Telefon, Gruendungsjahr, Team) | Muessen verifiziert sein. AI halluziniert. |
| **Texte** | Handgeschrieben, branchenspezifisch, Schweizer Tonalitaet. CC schreibt aus verifizierten Daten. |
| **Team-Fotos** | AI-Personen wirken unecht. Echte Fotos oder keine. |
| **Bewertungen** | Nur echte Google Reviews. Nie erfunden. |
| **Logos** | Betriebseigene Logos oder Firmenname als Wordmark. |

## 9. Was auf KEINEN Fall nach Template riechen darf

### Rang 1: HERO (hoechste Prioritaet)

Der Hero ist der erste Eindruck. Wenn zwei Heros gleich aussehen → Template erkannt.

**Loesung:** 3 verschiedene Hero-LAYOUTS (Split/Center/Classic) + Banana-Bild (unique pro Betrieb) + profilbasierte Farbtemperatur. Das reicht. Video-Loop ist Bonus.

### Rang 2: SERVICE-PRAESENTATION (zweithöchste Prioritaet)

Aktuell: JEDE Site hat identische 3-Col Service-Cards. Das ist der zweitstaerkste Fingerabdruck.

**Loesung:** 3 Service-Layouts:
- SUBSTANZ: 2-Col mit grossem Bild links
- VERTRAUEN: Stacked List (vertikal, ein Service pro Zeile)
- PRAEZISION: 3-Col Grid (aktuell, bewaehrt fuer breite Aufstellung)

### Rang 3: REVIEW-INSZENIERUNG

Aktuell: JEDE Site hat identisches 3-Karten-Review-Grid auf grauem Hintergrund.

**Loesung:** 3 Review-Stile:
- SUBSTANZ: 1 grosses Zitat prominent (emotional, persoenlich)
- VERTRAUEN: Horizontal-Scroll Carousel
- PRAEZISION: 3-Col Grid (wie aktuell)

### Rang 4: SCROLLRHYTHMUS

Aktuell: Section → Gap → Section → Gap. Immer gleich.

**Loesung:** framer-motion Scroll-Animationen. 3 Animations-Familien:
- SUBSTANZ: Fade-up (langsam)
- VERTRAUEN: Slide-in von den Seiten
- PRAEZISION: Scale-in (Cards wachsen)

### Was NICHT nach Template riechen muss (weil es niemand vergleicht)

- Footer (alle Footers sehen gleich aus, branchenweit)
- Contact-Section (Adresse + Karte ist Standard)
- Impressum/Datenschutz (rechtlich vorgegeben)
- Nav-Struktur (Logo + Links + CTA ist Industriestandard)

## 10. Produktionslogik fuer 3 Websites pro Tag

### Pipeline pro Betrieb (Ziel: 60 Min CC + 5 Min Founder)

```
VORBEREITUNG (15 Min CC):
  1. prospect_card.json validieren (Gates V1-V3 muessen PASS sein)
  2. Profil zuweisen (Regel → SUBSTANZ/VERTRAUEN/PRAEZISION)
  3. Website des Betriebs crawlen (Bilder + Fakten)

AI-GENERATION (10 Min CC, wartet auf API):
  4. Banana: 5 Hero-Kandidaten (profilbasierter Prompt)
  5. Banana: 1 Atmosphaere-Bild (lokales Szenario)

FOUNDER-REVIEW (5 Min Founder):
  6. Founder waehlt Hero (1 aus 5)
  7. Founder gibt "Go"

ASSEMBLY (25 Min CC):
  8. CustomerSite Config schreiben (Theme-Objekt + Content aus Crawl)
  9. Bilder platzieren (Hero + Service-Bilder aus Crawl + Atmosphaere)
  10. Build + Deploy

QA (10 Min CC):
  11. Mobile-First Visual Check
  12. Wizard E2E
  13. Lighthouse >= 85
  14. Alle Fakten korrekt (Name, Adresse, Telefon, Gruendungsjahr)

TOTAL: 60 Min CC + 5 Min Founder + CHF 0.24 API
```
