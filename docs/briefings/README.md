# Briefings

Ad-hoc Inputs, Transkripte und Kontext-Dokumente.

## Namenskonvention

```
YYYY-MM-DD__thema-kurz.md
```

## Regeln

- Jedes Briefing ist ein eigenständiges Dokument mit Datum und Quelle.
- Keine Tasks hier — Tasks gehören ins OPS_BOARD.md.
- Wenn aus einem Briefing eine Entscheidung folgt → `docs/decisions/DEC-xxx.md` anlegen.
- Template: `docs/templates/RESEARCH.md` (für strukturierte Inputs).
