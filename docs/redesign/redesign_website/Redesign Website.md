# Redesign Website

## Warum es diese Datei gibt

Diese Datei ist die Arbeitsgrundlage für das nächste Redesign von `flowsight.ch`.

Sie existiert aus einem klaren Grund:

**Die aktuelle Website schafft noch nicht den Wert, den FlowSight im Vertrieb eigentlich schaffen müsste.**

FlowSight ist inzwischen deutlich stärker als die Website. Das Produkt, die Gold-Contact-Logik, die persönliche Founder-Ansprache, die Video-Strategie, die Pain-Type-Logik und die operative Tiefe sind weiter als das, was die Website heute transportiert.

Dadurch entsteht eine Lücke:

- Das persönliche Video kann stark sein.
- Die persönliche Ansprache kann stark sein.
- Die Testlogik kann stark sein.
- **Aber wenn der Betrieb danach auf die Website klickt und dort keinen klaren roten Faden erlebt, sinkt Vertrauen, Verständnis und Conversion.**

Die Website ist nicht der primäre Verkäufer. Sie ist:

1. **Vertrauensverstärker** nach dem persönlichen Outreach
2. **Einordnungsmaschine** („Was ist FlowSight überhaupt?")
3. **Proof-Verstärker** („Das ist ein echtes System, kein Bastelprojekt.")
4. **Reibungsreduzierer** für den nächsten Schritt

Diese Datei soll deshalb die Grundlage sein, um das Redesign **nicht aus Produktperspektive**, sondern **aus Prospect-Perspektive** sauber aufzusetzen.

---

## Ziel des Redesigns

Die Website muss einem Handwerksbetrieb in kurzer Zeit diese 4 Dinge glasklar vermitteln:

1. **Was ist FlowSight?**
2. **Was bringt es meinem Betrieb konkret?**
3. **Wie läuft es im Alltag?**
4. **Was mache ich als Nächstes?**

Und zwar so, dass es:

- **mobile first** funktioniert
- für **Schweizer Handwerksbetriebe allgemein** anschlussfähig ist
- trotzdem aus dem **starken SHK-Kern** heraus glaubwürdig bleibt
- **high-end** aussieht, aber nicht wie schönes Chaos
- **kurz, klar, ruhig und hochwertig** wirkt
- nicht wie eine SaaS-Seite, sondern wie die Website eines ernsthaften Systems

## Aktueller Stand

### Positiv

- Navbar wurde bereits verbessert (`Live erleben` statt `Testen`)
- Hero geht sprachlich stärker in Richtung Leitsystem
- Grundgedanke „vom ersten Kontakt bis zur Bewertung" ist richtig
- Pricing und FAQ sind grundsätzlich vorhanden
- Ablaufgedanke ist inhaltlich bereits angelegt

### Negativ

Die Seite hat aktuell **keinen sauberen roten Faden**.

Sie wirkt an mehreren Stellen wie:

- teilweise SHK-Landingpage
- teilweise allgemeine Produktseite
- teilweise Tool-/Feature-Seite
- teilweise Vertriebsseite
- teilweise Demo-/Mockup-Sammlung

Das Problem ist **nicht**, dass jede einzelne Section schlecht ist.
Das Problem ist:

**Die Seite wirkt nicht wie ein bewusst geführtes Betriebserlebnis.**

Es fehlt:

- eine saubere Dramaturgie
- ein einheitliches visuelles Prinzip
- eine klare Nutzerführung
- eine stringente Übersetzung von Gold Contact auf Website-Ebene
- eine konsequente Prospect-Brille

# Grundlogik: Wie die Website wahrgenommen wird

## Der tatsächliche Nutzungskontext

Die Website wird in den meisten relevanten Fällen **nicht kalt** entdeckt.

Die wahrscheinlichere Reihenfolge ist:

1. Prospect erhält persönliche E-Mail / Video / Impuls
2. Prospect klickt auf `flowsight.ch`
3. Prospect prüft in Sekunden:
   - Ist das seriös?
   - Verstehe ich das?
   - Ist das für meinen Betrieb relevant?
   - Will ich weitergehen?

Die Website ist also **nicht der erste Verkauf**, sondern **die zweite Vertrauensstufe**.

Das ist entscheidend für Struktur und Copy.

### Konsequenz

Die Website muss nicht alles erklären.
Sie muss **schnell Vertrauen, Klarheit und Orientierung** schaffen.

# Die Prospect-Brille

## Wen sprechen wir an?

Nach außen:

- Schweizer Handwerksbetriebe

Innen strategisch:

- SHK bleibt Kernmarkt
- die öffentliche Website soll aber nicht so eng gebaut sein, dass bei Elektriker, Dach, Garage oder ähnlichen Gewerken sofort wieder ein strategischer Bruch entsteht

## Was will ein Handwerksbetrieb auf der Website nicht?

- keine langen Texte
- keine leeren Werbesprüche
- keine technische Architektur-Erklärung
- keine unklare Software-Sprache
- keine Feature-Liste ohne Zusammenhang
- keine schönen Screens, die nicht erklären, was sie bringen
- keine Zeitverschwendung

## Was will ein Handwerksbetrieb auf der Website?

- schnell verstehen, worum es geht
- sofort sehen, was das im Alltag bringt
- erkennen, dass hier Ordnung statt Chaos entsteht
- spüren, dass das System echt und professionell ist
- einen klaren nächsten Schritt sehen

## Mobile First oder nicht?

**Ja, klar mobile first.**

Warum:

- Inhaber / Meister schauen oft mobil
- wenig Zeit
- wenig Geduld
- viel Kontextwechsel
- Website wird oft zwischen Baustelle, Auto, Pause oder Feierabend geöffnet

### Konsequenzen aus Mobile First

- kurze Headline-Struktur
- kurze Subline-Struktur
- keine Textwände above the fold
- große Touch-Ziele
- schneller visueller Beweis
- klare CTA-Hierarchie
- visuelle Ruhe
- Section-Rhythmus statt Content-Matsch

**Die Website darf nicht zeigen, was FlowSight alles hat. Sie muss zeigen, wie ein Betrieb mit FlowSight sauber läuft.**

Das ist die Leitlinie für das gesamte Redesign.
