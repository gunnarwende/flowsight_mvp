# docs/_archive — Historie, nicht aktiv

Hierher wandern Dokumente, deren **einzigartiges Wissen bereits in eine kanonische Datei gesichert** wurde (siehe `docs/_consolidation_plan.md`, Abschnitt 2) oder die durch eine aktuellere Quelle abgelöst sind.

Regeln:
- **Nur Historie. Nicht editieren, nicht aus dem Archiv heraus verlinken** als wäre es aktuell.
- Nichts hier ist die Wahrheit über den Live-Stand — die liegt in den kanonischen Docs (siehe `docs/INDEX.md`) bzw. im System (Supabase/Retell/Cockpit).
- Git-Historie bleibt ohnehin erhalten; das Archiv hält die Dateien nur auffindbar, ohne den aktiven Lesepfad zu belasten.

Gruppen-Unterordner spiegeln die Herkunft (z. B. `redesign/`, `gtm-website/`, `gtm-legacy/`, `pipeline-history/`, `customers-*`, `gtm-onboarding/`).
