# Demo Practice — Workflow

## Nach jeder Uebung

1. **Teams-Transkript exportieren** (Meeting > Transkript > .vtt oder .docx)
2. **Transkript + Notizen** in den Datums-Ordner legen
3. **Selbsteinschaetzung** schreiben (3 Saetze: gut / unsicher / aendern)
4. **CC Review anfragen:** Transkript + Notizen an Claude Code schicken

## Ordnerstruktur

```
practice/
  2026-03-05_solo/           <- Erste Solo-Uebung
    transcript.vtt
    notes.md
  2026-03-08_kumpel-1/       <- Demo mit Kumpel 1
    transcript.vtt
    notes.md
    review.md                <- CC Feedback
  2026-03-09_kumpel-2/       <- Demo mit Kumpel 2
    transcript.vtt
    notes.md
    review.md
```

## Notizen-Template (notes.md)

```markdown
# Demo Practice — [Datum]

## Setup
- Typ: Solo / Mit Partner
- Partner: [Name]
- Dauer: [Min]

## Selbsteinschaetzung
- **Gut:** ...
- **Unsicher:** ...
- **Naechstes Mal anders:** ...

## Partner-Feedback (falls vorhanden)
- ...
```

## Fortschritt tracken

Nach jedem Review notiere ich dir:
- Filler-Woerter Trend (weniger/mehr?)
- Pitch-Fluss (natuerlicher?)
- Schwachstellen (gefixt?)
- Naechster Fokus-Punkt
